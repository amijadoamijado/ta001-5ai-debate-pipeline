## 4. 検証仕様（差分）

### 4.1 4段階フォールバック戦略の検証

design.md Section 7.3に定義されたフォールバック戦略が、各コマンドに正しく実装されていることを確認する。

**4段階フォールバック**:

```
レベル1: MCP経由で接続試行
  ↓ 失敗
レベル2: REST API直接呼び出し（環境変数にキーがあれば）
  ↓ 失敗
レベル3: WebSearchツール（Claude Code組み込み）で代替
  ↓ 失敗
レベル4: 訓練データの知識のみで続行（ログに警告記録）
```

### 4.2 テストシナリオ1: 全APIキー設定済み

**環境設定**:
```
BRAVE_API_KEY=brv_xxxxx  ← 設定済み
TAVILY_API_KEY=tvly_xxxxx ← 設定済み
EXA_API_KEY=exa_xxxxx     ← 設定済み
```

**検証項目**:

| # | 検証内容 | 期待結果 |
|---|---------|---------|
| 1 | Phase 0でBraveとTavilyが使い分けられる設計になっているか | 用途別に使い分けルールが明記 |
| 2 | Phase 3でBraveとExaが使い分けられる設計になっているか | 反証検索にBrave、類似パターン検索にExa |
| 3 | Phase 3.5で全3ツールが使い分けられる設計になっているか | Brave（広範）、Tavily（構造化）、Exa（セマンティック） |
| 4 | REQ-010.3のツール使い分けルールに準拠しているか | 検索目的ごとに最適ツールが選択される |

### 4.3 テストシナリオ2: 一部APIキー未設定

**環境設定例（Exa未設定）**:
```
BRAVE_API_KEY=brv_xxxxx  ← 設定済み
TAVILY_API_KEY=tvly_xxxxx ← 設定済み
EXA_API_KEY=              ← 未設定
```

**検証項目**:

| # | 検証内容 | 期待結果 |
|---|---------|---------|
| 1 | Exa未設定時にPhase 3がWebSearchで代替する設計になっているか | WebSearchフォールバック指示が記述 |
| 2 | Exa未設定時にPhase 3.5がWebSearchで代替する設計になっているか | WebSearchフォールバック指示が記述 |
| 3 | フォールバック時にログに警告が記録される設計になっているか | 「{ツール名} APIキー未設定、WebSearchで代替」の記録指示 |
| 4 | パイプラインが停止しないことが保証されているか | エラーではなくスキップ/代替 |

**追加テストパターン**:

| パターン | 未設定ツール | 影響フェーズ |
|---------|------------|------------|
| パターンA | Brave未設定 | Phase 0, Phase 3, Phase 3.5 |
| パターンB | Tavily未設定 | Phase 0, Phase 2, Phase 3.5 |
| パターンC | Exa未設定 | Phase 3, Phase 3.5 |
| パターンD | Brave+Exa未設定 | Phase 0, Phase 3, Phase 3.5 |

### 4.4 テストシナリオ3: 全APIキー未設定（Graceful Degradation）

**環境設定**:
```
BRAVE_API_KEY=              ← 未設定
TAVILY_API_KEY=             ← 未設定
EXA_API_KEY=                ← 未設定
```

**検証項目**:

| # | 検証内容 | 期待結果 |
|---|---------|---------|
| 1 | パイプラインが停止せずWebSearchで続行する設計になっているか | Graceful Degradation |

| 3 | ログに「全リサーチツールAPIキー未設定」警告が記録される設計か | 警告メッセージの記述確認 |
| 4 | Phase 3.5（歴史的検証）が訓練データ知識のみで続行可能か | 訓練データフォールバック指示が記述 |
