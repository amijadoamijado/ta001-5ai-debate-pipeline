# .handoff/AGENTS.md - Codex レビュー用ルール

このファイルは、**Codexをコードレビュー担当として使用する場合**の指示を定義します。

## 役割の明確化

| ファイル | 役割 |
|---------|------|
| `AGENTS.md`（プロジェクトルート） | Codex全体の設定・AI Coordination・Work Order Review |
| `.handoff/AGENTS.md`（このファイル） | コードレビュー専用の4段階手順 |

Codexの全体設定については `AGENTS.md` を参照してください。

---

## 前提条件

**必須**: レビュー前に、以下の品質ゲートを通過していること

```bash
npm run build  # 成功している
npm test       # 全パスしている
npm run lint   # エラーがない
```

品質ゲートを通過していないコードはレビューしません。

---

## レビュー手順（4段構成）

### 段1: 仕様整合性と破壊的変更の有無

**観点**
- 要件（.kiro/specs/*/requirements.md）と一致しているか
- 破壊的変更があれば、それは意図的か
- 既存のインターフェースを変更していないか

**出力形式**
```markdown
## 段1: 仕様整合性

### 逸脱の可能性
- [該当箇所]: 何が問題か

### 破壊的変更
- [ある/ない]
- [ある場合] 変更内容と影響範囲

### 読むべき関連ファイル
- [ファイルパス]: 確認すべき点

### 追加で必要な情報
- [不足している情報]
```

### 段2: 正しさと境界条件

**観点**
- バグの可能性
- エッジケースの考慮
- null/undefined の扱い
- エラーハンドリング

**出力形式**
```markdown
## 段2: 正しさと境界条件

### バグ候補
| 重大度 | 場所 | 問題 | 再現手順 |
|--------|------|------|----------|
| High | `src/auth.ts:45` | null参照 | user=nullの場合 |

### 修正案
- [バグごとの修正内容]
```

### 段3: セキュリティと運用

**観点**
- 危険な操作（eval、動的実行等）
- 認証・認可の不備
- 秘密情報のハードコード
- ログ出力の問題
- 権限チェック

**出力形式**
```markdown
## 段3: セキュリティと運用

### 危険箇所
| 重大度 | 場所 | 問題 | 軽減策 |
|--------|------|------|--------|
| High | `src/auth.ts:12` | ハードコードされた鍵 | 環境変数に移動 |

### ログ・権限の観点
- [問題点]
```

### 段4: 品質（可読性、重複、テスト）

**観点**
- 可読性（命名、構造）
- 重複コード
- テストカバレッジ
- テストの質

**出力形式**
```markdown
## 段4: 品質

### リファクタ提案
- [箇所]: 何をどう改善すべきか

### 追加テスト案
- [不足しているテストケース]
```

---

## 重大度分類

| レベル | 基準 | 例 |
|--------|------|-----|
| **Critical** | セキュリティホール、データ消失の可能性 | 秘密鍵漏洩、SQLインジェクション |
| **High** | 機能不全、明らかなバグ | null参照、論理エラー |
| **Medium** | 境界条件の考慮漏れ | エッジケースで失敗 |
| **Low** | 可読性、命名、重複 | 変数名が不明瞭 |

---

## レビュー結果のまとめ

**必須出力**:
1. 重大度ごとの件数
2. 各問題の根拠、影響範囲、修正案
3. 追加テスト案

```markdown
## レビューまとめ

| 重大度 | 件数 |
|--------|------|
| Critical | 0 |
| High | 2 |
| Medium | 5 |
| Low | 3 |

## 推奨アクション
- [ ] Critical/High の修正を優先
- [ ] 修正後、再レビューが必要か

## 追加テスト案
- [テストケース1]
- [テストケース2]
```

---

## Codexへの指示

1. **段1から順に実施してください**
2. **各段の結果を即座に出力してください**（全段終了後の出力は不要）
3. **問題がない場合でも「問題なし」と明記してください**
4. **修正案は具体的なコードレベルで提示してください**
